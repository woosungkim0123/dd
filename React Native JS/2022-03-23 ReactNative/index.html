React Native 개념 및 브릿지 통신
ReactNative
2022.03.01.

개념

React Native App에서 가장 중요한 부분은 Bridge(브릿지)들을 통해서 코드가 운영체제와 통신할 수 있도록 하는 인프라 시설이다


이것들이 React Native App을 구성하고 있다



React Native App을 다운받을 때 Javascript 코드만 다운받는 것이 아니라 사진에 보이는 모든 기본 시설이 있는 앱을 다운 받는 것이다


기본 시설들이 Javascript 코드가 운영체제와 이야기할 수 있도록 해준다


그래서 Java와 Xcode가 필요한 것이다. 우리는 웹을 만드는 것이 아닌 앱을 만드는 것이니까

React와의 차이

React JS로 웹사이트를 만들 때는 JSX로 HTML을 만들고 평범한 JS로 바꾸고 브라우저가 실행시켜 주는 구조이다.


그러나 React Native는 앱 안에 있는 브라우저가 아니다. 브라우저가 없음


React Native는 번역기라고 생각하면 편하다.


React Native는 인터페이스로 우리와 운영 체제(ios, 안드로이드) 사이에 있다


React Native로 코드를 짜면 java안드로이드 or ios 코드로 번역이 된다.


예시로 React Native로 Button 컴포넌트를 만들면 React Native가 ios, 안드로이드에게 버튼을 그려달라고 요청하게 된다.


React Native는 버튼을 만들지 않고 ios랑 안드로이드에 메세지를 보내서 버튼을 만들어 달라고 요청하는 것 뿐이다

이벤트 예시를 통한 메세지 전달 과정

클릭시 배경이 빨강으로 바뀌는 예시

1. 이벤트 클릭

사용자가 화면에서 버튼을 누름


이벤트는 Native(ios, android)에 기록이 된다.


native가 화면을 통제하기 때문(감지하는 코드가 있음)

2. 데이터 수집

ios와 안드로이드가 이벤트에 대한 정보를 수집하게 된다


화면 어디에 이벤트가 발생했는가, 눌러진 시간은 얼마인가 등등

3. JSON 메세지

React Native는 그 정보를 가지고 JSON 메세지를 생성한다.

4. 메세지 받기

자바스크립트의 코드로 메세지를 받는다(이벤트를 기다리고 있음)


여기까지 정리하면 이벤트가 발생하면 IOS와 안드로이드는 Bridge를 통해서 Javascript에 메세지를 전달한다.

5 ~ 8

Javascript에서 배경색을 빨강색으로 바꾸라는 메세지를 Native 운영체제로 보냄


운영체제에서 배경색을 빨강색으로 바꿈


우리는 단지 메세지를 주고 받는 것임


운영체제로부터 버튼을 눌렀다는 메세지를 받으면 배경색을 바꿔라 라고 답변하는 것

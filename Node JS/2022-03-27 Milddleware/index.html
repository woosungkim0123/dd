개념

중간에 있는 소프트웨어


브라우저가 request하면 서버가 응답하기 전에 그 사이에 middleware가 있음

const privateMiddleware = (req, res, next) => {
  const url = req.url
  if (url === '/protected') {
    return res.send('<h1>Not allowed</h1>')
  }
  next() // 다음 함수를 호출해준다
}
const handleHome = (req, res) => {
  // 사실 이 부분에서도 next 사용 가능
  return res.send('1')
  // res.end : 끝내는거
  // res.send : 보내는거
}

app.get('/', privateMiddleware, handleHome)
// '/' 부분이 route
// '/'로 get요청이 오면 express는 콜백함수에 req object res object를 넣어줌
// 뒤에 함수를 handler(controller)라고 함
// 여기서 privateMiddleware과 handleHome은 컨트롤러
// 사실 모든 컨트롤러(handler)는 미들웨어임
// 마지막에 응답을 하는 컨트롤러를 미들웨어라 부르지 않는 것일뿐

컨트롤러에는 총 3개의 인자(req, res, next)가 있고 next는 미들웨어에서 다음 함수로 이동할 때 사용한다.

// app.use는 global middleware
// 순서가 중요(맨 위부터 차례대로 실행되기 때문)
app.get('/', handleHome) // 미들웨어가 적용되지않음

morgan 함수를 호출하면 설정한대로 middleware를 return 해줌

import morgan from 'morgan'
const logger = morgan('common') // common은 설정부분
app.use(logger) // 미들웨어로 호출
